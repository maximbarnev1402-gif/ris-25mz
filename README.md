# Сеть Хэмминга — Коррекция опечаток

> *Вдохновлено романом «Цветы для Элджернона» (Дэниел Киз, 1966)*
> Чарли Гордон пишет "becaus" вместо "because" и т.д. — я решил помочь найти ему по неправильному написанию слова правильную форму из словаря с помощью программы.

## Как работает

Каждое слово кодируется в **бинарный вектор из 62 бит**:

| Биты | Признак | Пример для «was» |
|------|---------|-----------------|
| 0–25 | Наличие букв a–z | w=1, a=1, s=1 |
| 26–51 | Первая буква | бит 48 (w) = 1 |
| 52–61 | Длина слова | бит 54 (длина 3) = 1 |

Сеть состоит из двух слоёв:

```
Слой 1: y_i = W·x + b = n − d(x, эталон_i)   # чем ближе — тем выше балл
Слой 2: конкуренция нейронов (MAXNET)
```

## Ограничения

**Словарь фиксирован** — если правильного слова нет в `DICTIONARY`, сеть не найдёт его. Расширяется добавлением строк в список, повторное обучение не нужно.

**Порядок букв не учитывается** — «stop», «tops», «pots» дают одинаковые биты 0–25. Неоднозначность разрешается через первую букву (биты 26–51).

**Короткие слова конфликтуют** — «sed» → «see» вместо «said»: расстояние до «see» (d=1) меньше, чем до «said» (d=3), хотя по смыслу нужно «said». Биграммы в кодировке устранили бы эту ошибку.

**Edit-distance ≠ Hamming-distance** — тест pyspellchecker (160k слов) как источника кандидатов дал точность 21%: его кандидаты строятся по редакционному расстоянию, а опечатки Чарли — фонетические замены, которые редакционно далеки от правильного слова.
